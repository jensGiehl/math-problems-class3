# Gemini Coding Guidelines

## Project Context

*   **Domain**: The project is about generating math exercises for 3rd-grade students.
*   **Output**: The final output should be a PDF file in A4 format.
*   **Layout**: Ensure there is enough space for students to fill in the answers. The presentation of the exercises should be visually appealing.

## Coding Style

1.  **Clean Code**: Write clean, readable, and maintainable code.
2.  **OOP**: Follow Object-Oriented Programming principles.
3.  **No Comments**: Do not include implementation comments in the code.
4.  **Javadoc**: Provide Javadoc for classes and methods in English.
5.  **Logging**: Use `java.util.logging` for logging. Log messages must be in English.
6.  **Update README**: Update the `README.md` file to reflect changes.
7.  **JUnit Tests**: Write unit tests using JUnit and AssertJ.
8.  **Static Imports**: Use static imports where appropriate to improve readability.
9.  **Imports**: Always import classes. Do not use fully qualified names (e.g., import `java.util.Objects` instead of using `java.util.Objects` in the code).
